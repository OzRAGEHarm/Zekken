# v1.0.2 (March 30, 2026)

- Parser: Fixed empty-parameter functions using `||` so `func name || { ... }` parses correctly (instead of treating `||` as boolean OR).
- Parser: Extended pipe handling so `||` works consistently in pipe-delimited contexts (function declarations, lambda declarations, and `=> ||` zero-argument calls).
- Language: Added support for `in` operator with if statements.
  - `value in array` checks membership.
  - `"key" in obj` checks whether a key exists.
  - `"sub" in "string"` checks substring existence.
- VM parity: Implemented `in` in VM mode (opcode + runtime support) so it behaves the same as evaluator mode.
- Lint/runtime correctness: Fixed false `Reference Error` reports for locals declared inside function bodies in evaluator mode (the pre-execution lint pass now models sequential `let` declarations inside functions/lambdas).
- Lint/runtime correctness: Fixed block-scope linting so locals declared inside `while`/`if`/`try` blocks are visible for later statements in the same block during lint (prevents false "Variable not found" errors).
- CLI error output: Fixed function declaration highlighting so `func` is colored like a keyword, while the function name is highlighted separately (instead of both being the same color).
- Docs: Added `in` operator examples to Syntax Basics for `if value in collection { ... }` style conditions.
- VS Code/LSP: `@parse_json` now appears in `@...` completions and inserts the correct call snippet (`@parse_json => |...|`).
- VS Code/LSP: Reduced noisy keyword completions.
  - Statement keywords only suggest at statement-start (except `in`).
  - `else` is only suggested after a closing `}` (same line or previous non-empty line).
  - No completions inside strings/comments.
- Language: Added `array.remove => |value|` (removes the first matching element, mutates the array, returns the removed value).
- VS Code/LSP: Added `array.remove` to array method completions/snippets.
- Docs: Documented `array.remove` in Types -> Array Methods with an example.
