<h1>Release Development Log - Zekken Programming Language</h1>
<h3>Version 1.0.0 | (February 26, 2026 - March 26, 2026).</h3>

</div>

### Release Development Build #1 (February 26, 2026) | v1.0.0:
- Started the release-focused development push (performance + usability + tooling)
- Removed `--no-lint` from `zekken run` (it didn’t provide meaningful benchmarking wins)
- Bytecode VM: started the design/prototyping workstream (execution speed without losing Rust memory safety)
- Began collecting stress tests and real-world-style workloads to guide optimization work

### Release Development Build #2 (March 1, 2026) | v1.0.0:
- Bytecode VM: first working prototypes executing a meaningful subset of the language
- Began pushing validation work out of hot loops (avoid repeating expensive checks while executing)
- Tightened runtime hot paths (loops and repeated environment/value operations)

### Release Development Build #3 (March 5, 2026) | v1.0.0:
- Added benchmark suite under `benchmarks/` + runner scripts for comparing against Python
- Added CLI debug helpers:
  - `zekken debug tokens <file>` (lexer token stream)
  - `zekken debug ast <file>` (AST dump)
- Continued parity fixes between evaluator and VM behavior (keep both modes aligned)

### Release Development Build #4 (March 14, 2026) | v1.0.0:
- Expanded and documented standard library capabilities
- Added/expanded native modules used in tests and docs:
  - `encoding` (base64/hex/url encode/decode)
  - `path` (join/normalize/resolve/basename/dirname/extname/stem/is_abs/relative)
  - `http` (native-only HTTP client + server helpers; disabled on WASM)
- Improved OS utilities and process helpers in `os`

### Release Development Build #5 (March 20, 2026) | v1.0.0:
- Website documentation overhaul:
  - Improved wording and examples across Syntax Basics, Types, Functions, Libraries, and Errors
  - Added deep-linkable headers with hover `#` and click-to-copy links
  - Added per-function examples in the Libraries docs (examples appear directly under each function)
- Demo/Docs quality improvements to keep examples consistent with real Zekken call syntax (`=> |...|`)

### Release Development Build #6 (March 26, 2026) | v1.0.0:
- Added `zekken run --vm <file>` to run scripts on the bytecode interpreter (evaluator remains the default)
- Major performance work focused on hot paths (loops, indexing, math, and call dispatch)
- VS Code language support improvements:
  - Better hover data and module member signatures (using real Zekken call syntax)
  - Highlighting fixes for member/property access and function calls
  - Improved diagnostics behavior and usability
