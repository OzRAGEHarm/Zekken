# v1.0.1 (March 27, 2026)

## Build 1 (March 27, 2026)
- Fixed a bug where function bodies did not create function-scope environments that contained function parameters, so the interpreter tried to resolve function parameters as global variables.
- Fixed a bug with `Reference` Errors on the demo not being formatted/highlighted correctly
- Fixed a bug where built-ins such as `@println` could run without the `@` prefix so `println => ||` would execute, this does not happen anymore, calling built-in functions without the `@` prefix now results in a `Runtime Error`
- LSP: Fixed a bug where `let/const` declarations that did not end with a semicolon (`let x: int = 45`) would not be reported
- Fixed a bug with dot access such as `math.PI` or `object.a` where the property being accessed getting treated like a variable that must exist, they now get treated like actual properties
- Fixed computed object indexing where `object[key]` for example wouldn't evaluate `key` at runtime and was looking for a literal `key` property
- Fixed VM parity `for-in` loops on arrays, it now supports `for |item| in array` and `for |i, item| in array` (index + value)
- Fixed numeric literal member access, `99.cast => |"string"|` now parses correctly, it no longer parses `99.` as a float when the `.` is actually member access
- Fixed comments affecting parsing, `// ...` and `/* ... */` are now skipped by the lexer so they cannot break parsing around braces/expressions
- Fixed WASM math.random/rand_int/rand_choice/shuffle traps: WASM now uses `js_sys::Math::random()` instead of SystemTime, eliminating `Error: unreachable` in the web demo for those types of calls
- Improved reseved type-name identifier errors, `let obj: obj = {};` would give several syntax errors that did not relate to the code, it now emits a specific syntax error message "Type names are reserved and cannot be used as variable identifiers"
- Added an example of this in both [Syntax Basics](../../website/Docs/syntax-basics.html#variables-and-constants) and [Types](../../website/Docs/types.html#declared-types)
