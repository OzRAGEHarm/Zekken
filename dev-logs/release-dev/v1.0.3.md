# v1.0.3 (July 12, 2026)

- Parser: Fixed false `Reference Error` reports for `try/catch` catch parameters so the error variable is now recognized correctly inside catch blocks.
- Parser: Export declarations now require a block form, so multi-value exports must use `export { ... };`.
- Parser: Fixed a bug where no argument function calls `... => ||` were being treated as a binary OR (`||`) operator if the function call was in a loop's arguments or being used as a value for an assignment operator.
- Runtime: Reduced hot-path function call overhead for simple functions by relying on captured values instead of cloning the full parent environment on every call. This primarily targets benchmark-heavy workloads like `branch_calls`.
- Runtime: Function parent-usage analysis now includes default parameter expressions, so capture-only function calls can stay on the lighter execution path without losing access to needed values.
- Language: Added prefix logical NOT expression support (`!value`) and boolean NOT assignment handling in the runtimes, so patterns like `flip = !flip` evaluate correctly. (Also because i forgot to add this in the v1.0 release...)
