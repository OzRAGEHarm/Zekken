Getting Started

There are two ways to run Zekken: the current source-based workflow and the upcoming installer workflow.

Current: Build From Source

Build and run from the repository root:

Windows

.\scripts\build-cli.bat
# or .\scripts\build.bat cli

.\target\release\zekken.exe run main.zk
# or cargo run --release run main.zk

Linux/macOS

./scripts/build-cli.sh
# or ./scripts/build.sh cli

./target/release/zekken run main.zk
# or cargo run --release run main.zk

Future: Installer-Based Setup

A planned installer will install Zekken globally and update your environment so you can run it directly from any terminal. 

# Planned usage after installer setup
zekken run main.zk
zekken repl
zekken --help

Goal: no manual full-path executable calls, just zekken <cmd> once installation is complete.

Run And Verify

# Verify command is visible in your shell (planned installer flow)
zekken --version

# Run a file
zekken run main.zk

You can also test programs in the website demo.
